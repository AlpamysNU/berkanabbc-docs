    Авторизация по номеру телефона через Telegram Bot¶

    Концепция¶

    Вместо email/пароля — вход по номеру телефона через Telegram Bot. Пользователь вводит телефон на сайте, переходит по deep link в бота, тот присылает 4-значный код. Регистрация и вход — один процесс.

    Ключевые принципы: - Код хранится только в Django cache, не в БД. В dev сейчас используется FileBasedCache в backend/.cache, чтобы runserver и runbot видели один и тот же код. TTL = 10 минут, после успешного verify — удаляется. - Повторный вход: бот уже знает телефон → присылает код без запроса контакта. - PhoneAuthSession хранит только токен/телефон/telegram_id — не код.

    Полный Flow — первый вход¶

    Пользователь                  Сайт (Frontend)              Django (Backend)         Telegram Bot
     │                              │                              │                       │
     │── вводит телефон ──────────▶│                              │                       │
     │                              │── POST /auth/request-phone/ ▶│                       │
     │                              │   { phone: "+77771234567" }  │                       │
     │                              │                              │ создаёт               │
     │                              │                              │ PhoneAuthSession       │
     │                              │                              │ (token, phone, TTL)    │
     │                              │◀── { session_token, ─────────│                       │
     │                              │     deep_link, bot_username }│                       │
     │                              │                              │                       │
     │◀── показывает кнопку ────────│                              │                       │
     │    "Открыть @berkanatestbot" │                              │                       │
     │                              │                              │                       │
     │── нажимает кнопку ──────────▶ Telegram открывает бота ─────────────────────────────▶
     │                              │                              │  /start TOKEN          │
     │                              │                              │◀─ привязывает ─────────│
     │                              │                              │   telegram_id к сессии  │
     │                              │                              │                       │
     │                              │                              │ телефон неизвестен ──▶│ просит
     │                              │                              │                       │ поделиться
     │                              │                              │                       │ контактом
     │── нажимает "Поделиться ─────▶─────────────────────────────────────────────────────▶│
     │   номером телефона"          │                              │                       │ получает
     │                              │                              │                       │ контакт
     │                              │                              │                       │ ↓
     │                              │                              │                       │ проверяет
     │                              │                              │                       │ phone == session.phone
     │                              │                              │◀─ cache.set(key,code) ─│
     │                              │                              │   TTL=600s             │
     │◀─────────────────────────────────────────────────────────────────────────── код ───│
     │   "🔐 Ваш код: 4821"        │                              │                       │
     │                              │                              │                       │
     │── нажимает "Я получил код" ─▶│                              │                       │
     │── вводит "4821" ────────────▶│                              │                       │
     │                              │── POST /auth/verify-phone/ ──▶│                       │
     │                              │   { session_token, phone,    │                       │
     │                              │     code }                   │                       │
     │                              │                              │ cache.get(key) == code │
     │                              │                              │ cache.delete(key)      │
     │                              │                              │ session.is_used = True │
     │                              │◀── { access, refresh, ───────│                       │
     │                              │     is_new_user }            │                       │
     │◀── redirect /dashboard ──────│                              │                       │

    Повторный вход (телефон уже известен боту)¶

    /start TOKEN → бот проверяет TelegramUserMap.phone
             → phone совпадает с session.phone?
               ДА → cache.set(code, TTL=10min) → отправляет код напрямую
               НЕТ → запрашивает контакт (contact_handler)

    Пользователю не нужно повторно нажимать «Поделиться номером» при каждом входе.

    Модели¶

    PhoneAuthSession¶

    class PhoneAuthSession(models.Model):
    phone           = CharField(max_length=20)       # нормализованный: +77771234567
    deep_link_token = CharField(max_length=64, unique=True)
    telegram_id     = BigIntegerField(null=True)      # заполняется после /start TOKEN
    expires_at      = DateTimeField()                 # now + 10 min
    is_used         = BooleanField(default=False)
    created_at      = DateTimeField(auto_now_add=True)
    # поле code НЕ используется — код хранится только в кэше

    TelegramUserMap¶

    class TelegramUserMap(models.Model):
    telegram_id         = BigIntegerField(unique=True)
    telegram_username   = CharField(max_length=64, blank=True)
    telegram_first_name = CharField(max_length=64, blank=True)
    phone               = CharField(max_length=20, blank=True)  # заполняется при contact_handler

    UserProfile¶

    class UserProfile(models.Model):
    user             = OneToOneField(User)
    telegram_id      = BigIntegerField(unique=True)
    telegram_username = CharField(max_length=64, blank=True)
    phone            = CharField(max_length=20, blank=True)  # нормализованный телефон
    full_name        = CharField(max_length=200, blank=True)
    country, city    = CharField(...)
    role             = CharField(choices=['parent','specialist','admin'])
    profile_complete = BooleanField()   # auto: full_name AND country AND city
    preferred_language = CharField(choices=['ru','kz','en'])

    Кэш кодов¶

    # settings.py — общий файловый кэш для runserver + runbot в dev
CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.filebased.FileBasedCache',
        'LOCATION': BASE_DIR / '.cache',
    }
}

# Ключ: berkana:phone_auth:{session_token}
# TTL: 600 секунд (10 минут)
# Удаляется после успешного verify или по истечению TTL

    Важно: LocMemCache не подходит для текущего dev-flow, потому что runserver и runbot — разные процессы. При locmem бот кладёт код в память одного процесса, а API проверяет память другого процесса. Поэтому в dev выбран FileBasedCache. Для продакшна при multi-worker деплое нужен внешний кэш (Redis/Memcached).

    API Endpoints¶

    POST /api/auth/request-phone/¶

    // Request
{ "phone": "+77771234567" }

// Response 200
{
  "status": "session_created",
  "session_token": "3ovN5-FH8nm_Y5fl...",
  "deep_link": "https://t.me/berkanatestbot?start=3ovN5-FH8nm_...",
  "bot_username": "berkanatestbot",
  "expires_in_seconds": 600
}

// Response 429
{ "error": "Слишком много попыток. Подождите 10 минут." }

    POST /api/auth/verify-phone/¶

    // Request
{
  "session_token": "3ovN5-FH8nm_Y5fl...",
  "phone": "+77771234567",
  "code": "4821"
}

// Response 200
{
  "access": "eyJ...",
  "refresh": "eyJ...",
  "is_new_user": true,
  "profile_complete": false
}

// Response 400 — сессия не найдена
{ "error": "Сессия не найдена или истекла. Начните вход заново." }

// Response 400 — код истёк в кэше
{ "error": "Код истёк. Вернитесь в Telegram и запросите новый." }

// Response 400 — неверный код
{ "error": "Неверный код. Проверьте сообщение в Telegram." }

    POST /api/auth/refresh/¶

    // Request
{ "refresh": "eyJ..." }

// Response 200
{ "access": "eyJ..." }

    Runtime-фиксы 2026-05-04 21:17 +05¶

    • Frontend хранит phone/session_token/deep_link/bot_username в sessionStorage, чтобы refresh страницы не ломал ввод кода.
    • verify-phone переиспользует существующего пользователя phone_..., если предыдущая попытка создала auth_user, но не завершила профиль.
    • verify-phone ищет UserProfile по telegram_id, если профиль по телефону ещё не найден.
    • PhoneAuthSession.is_used=True и удаление кэша выполняются только после успешного получения/создания пользователя и профиля.

    Telegram Bot — логика handlers¶

    start_handler¶

    Условие Действие
    /start (без токена) Приветствие + инструкция
    /start TOKEN, токен не найден/истёк Сообщение об ошибке + ссылка на сайт
    /start TOKEN, телефон уже известен и совпадает Сразу генерирует код → кладёт в кэш → отправляет
    /start TOKEN, телефон неизвестен или не совпадает Запрашивает контакт (кнопка «Поделиться номером»)

    contact_handler¶

    1. Проверяет, что контакт принадлежит самому пользователю
    2. Нормализует телефон из контакта
    3. Сохраняет телефон в TelegramUserMap.phone
    4. Ищет активную PhoneAuthSession по telegram_id
    5. Проверяет session.phone == phone
    6. Генерирует код → cache.set(key, code, TTL=600)
    7. Отправляет код пользователю
    8. Не записывает код в БД

    Нормализация телефона¶

    def normalize_phone(raw: str) -> str:
    digits = re.sub(r'\D', '', raw)
    if digits.startswith('8') and len(digits) == 11:
        digits = '7' + digits[1:]
    if not digits.startswith('7') and len(digits) == 10:
        digits = '7' + digits
    return '+' + digits

# +7 777 123-45-67 → +77771234567
# 87771234567      → +77771234567
# 7771234567       → +77771234567

    Безопасность¶

    Угроза Защита
    Brute force Rate limit: 3 активных сессии на телефон за 10 мин
    Повторное использование кода cache.delete(key) сразу после успешного verify
    Истечение кода TTL кэша 10 мин, отдельно от PhoneAuthSession.expires_at
    Чужой телефон Контакт в Telegram = верифицированный номер (платформа Telegram)
    Подделка контакта contact.user_id == user.id проверяется в contact_handler
    Утечка кода Код никогда не пишется в БД, только в Django cache; dev: FileBasedCache, prod: Redis/Memcached
