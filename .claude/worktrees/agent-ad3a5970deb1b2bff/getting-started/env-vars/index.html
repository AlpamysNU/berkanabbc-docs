    Environment Variables¶

    Backend — backend/.env¶

    Copy backend/.env.example to backend/.env and fill in values.

    Variable Required Example Description
    SECRET_KEY ✅ django-insecure-... Generate: python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
    DEBUG — True Set False in production
    DATABASE_URL ✅ postgresql://user:pass@localhost:5432/berkana PostgreSQL only
    TELEGRAM_BOT_TOKEN ✅ 123456:ABC-... From @BotFather
    TELEGRAM_BOT_USERNAME ✅ YourBotName Without @
    TELEGRAM_WEBHOOK_URL Prod https://api.berkanabbc.com/api/telegram/webhook/ Set after deploy
    ALLOWED_HOSTS Prod api.berkanabbc.com,localhost Comma-separated
    CORS_ALLOWED_ORIGINS — http://localhost:5173 Frontend origin(s), comma-separated

    Frontend — .env.local¶

    Copy .env.example to .env.local.

    Variable Required Dev value Prod value
    VITE_API_URL ✅ http://localhost:8000/api https://api.berkanabbc.com/api

    Security notes¶

    Never commit secrets

    .env and .env.local are in .gitignore — never commit them. Use .env.example as a template.

    • SECRET_KEY must be at least 50 characters and unique per environment
    • TELEGRAM_BOT_TOKEN gives full control of the bot — rotate if exposed
    • When DEBUG=False, Django auto-enables HTTPS headers (HSTS, secure cookies, XSS filter)
