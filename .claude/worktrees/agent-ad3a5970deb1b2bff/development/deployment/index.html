    Deployment Guide¶

    Stack¶

    Компонент Технология
    Web-сервер Nginx
    App-сервер gunicorn (WSGI)
    База данных PostgreSQL 14+
    Статика Nginx раздаёт backend/staticfiles/
    Медиафайлы Nginx или S3 (не Django в production)
    Frontend Nginx раздаёт Vite dist/

    1. Настроить переменные окружения¶

    Установить все переменные из env-vars.md, обязательно:

    DEBUG=False
SECRET_KEY=<50+ символов, уникальный для prod>
DATABASE_URL=postgresql://berkana:pass@localhost:5432/berkana
ALLOWED_HOSTS=api.berkanabbc.com
CORS_ALLOWED_ORIGINS=https://berkanabbc.com
TELEGRAM_BOT_TOKEN=...
TELEGRAM_WEBHOOK_URL=https://api.berkanabbc.com/api/telegram/webhook/

    2. Backend¶

    source backend_env/bin/activate
pip install -r requirements.txt
cd backend
python manage.py migrate
python manage.py collectstatic --noinput

# Запустить gunicorn (3 воркера — отправная точка)
gunicorn config.wsgi:application \
  --bind 0.0.0.0:8000 \
  --workers 3 \
  --timeout 60 \
  --access-logfile /var/log/gunicorn/access.log

    3. Frontend¶

    echo "VITE_API_URL=https://api.berkanabbc.com/api" > .env.production
npm run build
# dist/ готов для раздачи через Nginx

    4. Nginx (пример конфигурации)¶

    # API backend
server {
    listen 443 ssl;
    server_name api.berkanabbc.com;

    location /static/ {
        alias /path/to/backend/staticfiles/;
        expires 30d;
    }

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# Frontend SPA
server {
    listen 443 ssl;
    server_name berkanabbc.com;

    root /path/to/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}

    5. Security headers¶

    При DEBUG=False Django автоматически включает:

    Заголовок Значение
    Strict-Transport-Security max-age=31536000; includeSubDomains; preload
    X-Frame-Options DENY
    X-Content-Type-Options nosniff
    Session/CSRF cookies Secure=True

    DRF rate limiting (всегда активен):

    Тип Лимит
    Анонимный 30 req/min
    Авторизованный 120 req/min
    Auth endpoints 5 req/min
