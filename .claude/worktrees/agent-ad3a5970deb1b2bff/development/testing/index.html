    Running Tests¶

    Backend¶

    # Все тесты screening app
DATABASE_URL=postgresql://user:pass@localhost:5432/berkana \
  backend_env/bin/python backend/manage.py test screening

# Только import тесты (27 тестов, TDD)
DATABASE_URL=... backend_env/bin/python backend/manage.py test screening.tests_import

# Только report generator тесты
DATABASE_URL=... backend_env/bin/python backend/manage.py test screening.tests_report_generator

# Verbose output
DATABASE_URL=... backend_env/bin/python backend/manage.py test screening -v 2

    Ключевые тест-файлы:

    Файл Что тестирует
    backend/screening/tests_import.py 27 тестов: docx-парсер, preview API, confirm API
    backend/screening/tests_report_generator.py Генерация отчёта, domain_scores, зоны
    backend/screening/tests.py Session/Answer/Report API endpoints

    Frontend¶

    npm run lint      # ESLint — 0 ошибок обязательно
npm run build     # TypeScript compile + Vite build — должен проходить

    Юнит-тестов фронтенда нет — тестирование через E2E (ручное):

    1. Войти через Telegram Bot
    2. Создать профиль пользователя + профиль ребёнка
    3. Начать скрининг → ответить на все вопросы → завершить
    4. Убедиться что отчёт показывает зоны (🟢🟡🔴) по доменам
    5. Скачать PDF → файл должен открываться и содержать водяной знак
