    Скрининговые модули и Алгоритм Скоринга¶

    Статус¶

    Контент прочитан из BERKANA IORR v3.3 и новых DOCX v3.4-v3.6. 0-6 и 7-12 месяцев уже импортированы. Модули 13-24, 25-36 и 37-48 месяцев нормализованы из трехъязычных DOCX в JSON и загружены через import_screening.

    ⚠️ Важное отличие от исходного плана: зоны оценки отличаются от placeholder-значений в wiki: - Зелёная: ≥ 75% (было 75%, совпадает) - Жёлтая: 40–74% (было 50–74% — ИСПРАВЛЕНО) - Красная: < 40% (было 0–49% — ИСПРАВЛЕНО)

    Как добавлять новый модуль — стандартный workflow¶

    Способ А: Admin UI (рекомендуется, без разработчика) ✅¶

    1. Войти как role=admin → перейти в /admin/screening/import
    2. Нажать «Скачать шаблон» → скачается berkana_module_template_v2.docx
    3. Заполнить три секции шаблона (RU + EN + KZ обязательны для каждого поля)
    4. Перетащить/выбрать файл → система покажет preview с ошибками/предупреждениями
    5. Если ошибок нет → нажать «Подтвердить импорт» → модуль добавлен в БД

    Формат шаблона berkana_module_template_v2.docx:

    Секция Содержание
    1. ИНФОРМАЦИЯ О МОДУЛЕ Название RU/EN/KZ, возраст от/до (мес), интервал повторения (дней)
    2. ВОПРОСЫ Код, Домен, Тип, Возраст от/до, Текст RU/EN/KZ, Подсказка RU/EN/KZ
    3. ПРАВИЛА СКОРИНГА Домен, Зона (typical/monitor/follow_up), Мин/Макс %, Интерпретация RU/EN/KZ, Рекомендация RU/EN/KZ

    Маркер первой строки: BERKANA_TEMPLATE_v2 (обязателен, иначе ошибка).

    Допустимые домены: motor_gross, motor_fine, speech, social, self_care, red_flag, early_reg, cognitive, sensory, family.
    Допустимые типы: berkana_4opt, yes_no, scale_1_5, multiple_choice, text.
    Домены red_flag и early_reg не участвуют в скоринге (include_in_score=False).

    Способ Б: Dev management command (только для разработчиков)¶

    # JSON-шаблон (legacy):
backend_env/bin/python backend/manage.py import_screening raw/my_new_module.json --clear

# Перегенерировать .docx шаблон:
backend_env/bin/python backend/manage.py create_screening_template

    Шаг: Проверить после импорта¶

    backend_env/bin/python backend/manage.py shell -c "
from screening.models import ScreeningModule
for m in ScreeningModule.objects.all():
    print(m.name_ru, m.age_min_months, m.age_max_months, m.questions.count(), 'вопросов')
"

    Workflow для трехъязычных DOCX v3.4-v3.6¶

    Исходные DOCX из raw/Tests/ не редактируются. Нормализация выполняется отдельной командой:

    backend_env/bin/python backend/manage.py generate_berkana_imports

    Команда читает RU/EN/KZ документы, извлекает вопросы из блоков БЛОК/BLOCK/*-БЛОК, берет text_* из секции "Как проверить / How to check / Қалай тексеруге болады" и help_* из "Почему важно / Why it matters / Неліктен маңызды". EN/KZ сопоставляются с RU-каноном по порядку внутри блоков, потому что коды локализованы (РФ, KM, НМ, М, Ф, К, Ә, ӨЗ, Т).

    Готовые JSON: - backend/raw/berkana_iorr_v3_4_13_24_months_import.json - backend/raw/berkana_iorr_v3_5_25_36_months_import.json - backend/raw/berkana_iorr_v3_6_37_48_months_import.json

    Импорт:

    backend_env/bin/python backend/manage.py import_screening backend/raw/berkana_iorr_v3_4_13_24_months_import.json --clear
backend_env/bin/python backend/manage.py import_screening backend/raw/berkana_iorr_v3_5_25_36_months_import.json --clear
backend_env/bin/python backend/manage.py import_screening backend/raw/berkana_iorr_v3_6_37_48_months_import.json --clear

    Стандарт JSON-шаблона (schema_version 1.0)¶

    Полный эталон: raw/screening_import_template_full.json

    Структура верхнего уровня¶

    {
  "schema_version": "1.0",
  "module": { ... },
  "scoring_rules": [ ... ],
  "questions": [ ... ]
}

    Секция module¶

    Поле Тип Обязательно Пример
    name_ru string ✅ "BERKANA IORR v3.3: 13-24 месяца"
    name_en string ✅ "BERKANA IORR v3.3: 13-24 months"
    name_kz string ✅ "BERKANA IORR v3.3: 13-24 ай"
    description_ru string рекомендуется Описание для интерфейса
    description_en string рекомендуется
    description_kz string рекомендуется
    age_min_months int ✅ 13
    age_max_months int ✅ 24
    rescreen_interval_days int ✅ 90
    source_docx string нет Имя исходного DOCX (для трассировки)

    Секция scoring_rules¶

    Массив по доменам. Каждый домен — объект с domain и массивом rules.

    Домены в rules: создаются по scored-доменам, реально присутствующим в модуле: motor_gross, motor_fine, speech, social, self_care, cognitive. red_flag и early_reg не скорятся.

    Каждое правило:

    Поле Тип Описание
    min_score int Нижний порог зоны (включительно), %
    max_score int Верхний порог зоны (включительно), %
    severity string "typical" / "monitor" / "follow_up"
    interpretation_ru/en/kz string Текст интерпретации для родителя
    recommendation_ru/en/kz string Рекомендация для родителя

    Стандартные пороги (можно переопределить в JSON): - typical: 75–100% - monitor: 40–74% - follow_up: 0–39%

    Секция questions¶

    Каждый вопрос:

    Поле Тип Обязательно Описание
    code string ✅ Уникальный код вопроса в модуле (M1, K2, РФ3)
    block string нет Название блока (отображается в опроснике как заголовок)
    domain string ✅ Домен из списка ниже
    question_type string ✅ Тип вопроса (обычно berkana_4opt)
    title_ru/en/kz string рекомендуется Короткий заголовок вопроса
    text_ru string ✅ Полный текст вопроса на русском
    text_en string ✅ Английский перевод
    text_kz string ✅ Казахский перевод
    help_ru/en/kz string нет Подсказка / научный комментарий
    options array для 4opt Варианты ответов (см. ниже)
    age_min_months int нет Показывать с этого возраста
    age_max_months int нет Показывать до этого возраста
    include_in_score bool ✅ true — включать в доменный балл
    sort_order int ✅ Порядок отображения (шаг 10)
    is_active bool ✅ true / false

    Допустимые домены:

    Код Название В балл?
    motor_gross Крупная моторика ✅
    motor_fine Мелкая моторика ✅
    speech Коммуникация и речь ✅
    social Социально-эмоциональное ✅
    self_care Самообслуживание и адаптивное поведение ✅
    red_flag Красные флаги ❌
    early_reg Ранние регуляторные маркеры ❌
    cognitive Когнитивное развитие ✅
    sensory Сенсорная интеграция ✅
    family Семья и анамнез ❌

    Варианты ответов options (стандарт berkana_4opt)¶

    [
  {"value": "yes",       "label_ru": "ДА",           "label_en": "YES",       "label_kz": "ИӘ",          "score": 10},
  {"value": "sometimes", "label_ru": "ИНОГДА",       "label_en": "SOMETIMES", "label_kz": "КЕЙДЕ",       "score": 5},
  {"value": "not_yet",   "label_ru": "ЕЩЁ НЕТ",     "label_en": "NOT YET",   "label_kz": "ӘЛІ ЖОҚ",    "score": 0},
  {"value": "not_tried", "label_ru": "НЕ ПРОБОВАЛИ", "label_en": "NOT TRIED", "label_kz": "БАЙҚАМАДЫҚ",  "score": null}
]

    score: null — ответ «Не пробовали» исключается из знаменателя при расчёте процента.

    Простой TXT-формат (только RU, без scoring_rules)¶

    Для быстрой проверки без переводов — raw/screening_import_template.txt:

    [motor_gross]
M1 | Текст вопроса для родителя | Короткое название | возраст_с_месяцев

    ⚠️ TXT-формат генерирует предупреждение о missing EN/KZ translations. Для продакшна использовать JSON.

    Домены оценки (фактические из документа)¶

    Каждый вопрос относится к одному из доменов:

    Домен Код Блок в документе
    Крупная моторика motor_gross Блок 2
    Мелкая моторика motor_fine Блок 3
    Коммуникация/Речь speech Блок 4
    Социально-эмоциональное social Блок 5
    Красные флаги red_flag Блок 1А — не в расчёте
    Ранние маркеры early_reg Модуль A — не в расчёте

    Примечание: Домены cognitive и sensory из исходного wiki-плана отсутствуют в документе v3.3 (0–6 мес). Возможно появятся в следующих модулях (12+ мес). Модель Question.domain нужно обновить.

    Типы вопросов¶

    1. yes_no (Да / Нет)¶

    Самый простой тип. Балл: Да = 1, Нет = 0 (или наоборот, зависит от вопроса).

    {
  "question_type": "yes_no",
  "text_ru": "Ребёнок самостоятельно держит голову?",
  "domain": "motor",
  "options": null
}
// Scoring: "yes" → score=1, "no" → score=0

    2. scale_1_5 (Шкала от 1 до 5)¶

    Для вопросов о частоте или степени выраженности.

    {
  "question_type": "scale_1_5",
  "text_ru": "Как часто ребёнок улыбается в ответ на ваши эмоции?",
  "domain": "social",
  "options": null
}
// 1 = никогда, 5 = всегда
// score = value (1-5), нормализуется при подсчёте

    3. multiple_choice (Выбор варианта)¶

    {
  "question_type": "multiple_choice",
  "text_ru": "Сколько слов произносит ребёнок самостоятельно?",
  "domain": "speech",
  "options": [
    {"value": "a", "label_ru": "Не произносит слов", "score": 0},
    {"value": "b", "label_ru": "1–5 слов", "score": 1},
    {"value": "c", "label_ru": "6–20 слов", "score": 2},
    {"value": "d", "label_ru": "Более 20 слов", "score": 3}
  ]
}

    Алгоритм скоринга¶

    1. Подсчёт балла ответа¶

    # При сохранении Answer
def calculate_answer_score(question, value):
    if question.question_type == 'yes_no':
        return 1 if value == 'yes' else 0
    elif question.question_type == 'scale_1_5':
        return int(value)  # 1-5
    elif question.question_type == 'multiple_choice':
        option = next(o for o in question.options if o['value'] == value)
        return option['score']

    2. Подсчёт доменного скора (при complete)¶

    def calculate_domain_scores(session):
    answers = Answer.objects.filter(session=session).select_related('question')
    domains = {}

    for answer in answers:
        domain = answer.question.domain
        if domain not in domains:
            domains[domain] = {'actual': 0, 'max': 0}
        domains[domain]['actual'] += answer.score
        domains[domain]['max'] += get_max_score(answer.question)

    # Нормализация в проценты
    return {
        domain: round((data['actual'] / data['max']) * 100)
        for domain, data in domains.items()
        if data['max'] > 0
    }

def get_max_score(question):
    if question.question_type == 'yes_no':
        return 1
    elif question.question_type == 'scale_1_5':
        return 5
    elif question.question_type == 'multiple_choice':
        return max(o['score'] for o in question.options)

    3. Интерпретация (ScoringRule lookup)¶

    def get_interpretations(module, domain_scores, language='ru'):
    result = {}
    for domain, score in domain_scores.items():
        rule = ScoringRule.objects.get(
            module=module,
            domain=domain,
            min_score__lte=score,
            max_score__gte=score
        )
        result[domain] = {
            'text': getattr(rule, f'interpretation_{language}'),
            'recommendation': getattr(rule, f'recommendation_{language}'),
            'severity': rule.severity
        }
    return result

    4. Overall level¶

    def get_overall_level(interpretations):
    severity_rank = {'typical': 0, 'monitor': 1, 'follow_up': 2}
    worst = max(
        interpretations.values(), 
        key=lambda x: severity_rank[x['severity']]
    )
    return worst['severity']

    5. Delta scores (повторный скрининг)¶

    def calculate_delta(current_report, previous_session):
    prev_report = Report.objects.get(session=previous_session)
    delta = {}
    for domain, score in current_report.domain_scores.items():
        prev_score = prev_report.domain_scores.get(domain, 0)
        delta[domain] = score - prev_score
    return delta
# +5 = улучшение ↑, -3 = снижение ↓, 0 = без изменений →

    Диапазоны ScoringRule (пример)¶

    Для каждого модуля и домена нужно 3 правила:

    Domain min max Severity Пример интерпретации (RU)
    motor 75 100 typical Моторные навыки соответствуют возрасту
    motor 50 74 monitor Моторное развитие требует дополнительного внимания
    motor 0 49 follow_up Рекомендована консультация специалиста по двигательному развитию

    Аналогично для cognitive, social, speech, sensory.

    Примеры вопросов для seed (Ранний возраст, 12–35 мес)¶

    Моторика¶

    1. Ребёнок самостоятельно ходит без поддержки? (yes_no)
    2. Ребёнок может подняться по ступенькам, держась за поручень? (yes_no)
    3. Насколько уверенно ребёнок манипулирует небольшими предметами (кубики, ложка)? (scale_1_5)
    4. Ребёнок может бросить мяч руками? (yes_no)
    5. Ребёнок рисует каракули карандашом? (yes_no)

    Когнитивное¶

    1. Ребёнок понимает простые инструкции ("дай", "покажи")? (yes_no)
    2. Ребёнок узнаёт знакомые предметы по картинке? (yes_no)
    3. Ребёнок может найти спрятанный предмет? (yes_no)
    4. Как часто ребёнок проявляет любопытство к новым предметам? (scale_1_5)
    5. Ребёнок имитирует действия взрослых (говорит по телефону, "читает")? (yes_no)

    Речь¶

    1. Сколько слов произносит ребёнок самостоятельно? (multiple_choice: 0/1-5/6-20/20+)
    2. Ребёнок понимает двухшаговые инструкции? (yes_no)
    3. Ребёнок использует жесты (машет рукой, указывает пальцем)? (yes_no)
    4. Насколько разборчива речь ребёнка для незнакомых людей? (scale_1_5)
    5. Ребёнок называет знакомых людей по имени? (yes_no)

    Социальные навыки¶

    1. Как часто ребёнок инициирует игровой контакт? (scale_1_5)
    2. Ребёнок проявляет привязанность к близким? (yes_no)
    3. Ребёнок реагирует на своё имя? (yes_no)
    4. Ребёнок играет рядом с другими детьми (параллельная игра)? (yes_no)
    5. Ребёнок проявляет эмпатию (реагирует на плач другого ребёнка)? (yes_no)

    Сенсорная интеграция¶

    1. Ребёнок нормально переносит прикосновения (одевание, объятия)? (yes_no)
    2. Ребёнок сильно реагирует на громкие звуки? (yes_no, обратная шкала)
    3. Насколько ребёнок чувствителен к новым вкусам и текстурам еды? (scale_1_5)
    4. Ребёнок ищет сенсорные ощущения (раскачивается, трогает всё)? (yes_no)
    5. Ребёнок нормально реагирует на яркий свет? (yes_no)

    Важные замечания¶

    • Все тексты вопросов и интерпретаций должны быть проверены специалистом (психологом, специалистом по развитию)
    • Обратные вопросы (сильная чувствительность = меньший балл) должны быть явно помечены при создании
    • Seed-данные — только пример структуры, не для клинического использования
    • Рекомендации должны использовать наблюдательный язык, не диагностический
