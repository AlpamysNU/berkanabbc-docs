    Интерактивная документация (Swagger UI)¶

    При запущенном backend сервере (только в режиме DEBUG=True) полная API документация доступна интерактивно:

    Инструмент URL Описание
    Swagger UI http://localhost:8000/api/schema/swagger-ui/ Интерактивное тестирование прямо в браузере
    ReDoc http://localhost:8000/api/schema/redoc/ Красивый read-only вид
    OpenAPI JSON http://localhost:8000/api/schema/ Raw OpenAPI 3.0 schema

    В production: заменить localhost:8000 на https://api.berkanabbc.com

    REST API Reference¶

    Base URL: http://localhost:8000/api/

    Все endpoints, кроме auth/request-phone/, auth/verify-phone/, auth/request-code/, auth/verify-code/ и telegram/webhook/, требуют заголовка:

    Authorization: Bearer <access_token>

    Access token TTL: 15 минут. Обновление через auth/refresh/.

    1. Авторизация (/api/auth/)¶

    POST /api/auth/request-phone/¶

    Основной auth flow. Принимает номер телефона, создаёт PhoneAuthSession, возвращает Telegram deep link.

    Права: AllowAny
    Rate limit: не более 3 активных сессий на один номер телефона

    // Request
{ "phone": "+7 777 123 45 67" }

// Response 200
{
  "status": "session_created",
  "session_token": "3ovN5-FH8nm_Y5fl...",
  "deep_link": "https://t.me/berkanatestbot?start=3ovN5-FH8nm_...",
  "bot_username": "berkanatestbot",
  "expires_in_seconds": 600
}

// Response 429
{ "error": "Слишком много попыток. Подождите 10 минут." }

    POST /api/auth/verify-phone/¶

    Шаг 2: пользователь вводит 4-значный код из Telegram → получает JWT.

    Права: AllowAny

    // Request
{
  "session_token": "3ovN5-FH8nm_Y5fl...",
  "phone": "+77771234567",
  "code": "1234"
}

// Response 200
{
  "access": "eyJ...",
  "refresh": "eyJ...",
  "is_new_user": true,
  "profile_complete": false
}

// Response 400
{ "error": "Invalid or expired code" }

    POST /api/auth/refresh/¶

    Обновление access токена.

    Права: AllowAny

    // Request
{ "refresh": "eyJ..." }

// Response 200
{ "access": "eyJ..." }

// Response 401
{ "error": "Token is invalid or expired" }

    Получить профиль текущего пользователя.

    Права: IsAuthenticated

  "first_name": "Иван",

  "full_name": "Иванов Иван Иванович",
  "display_name": "Иван Иванов",

  "email": "ivan@example.com",
  "country": "Казахстан",
  "city": "Алматы",
  "role": "parent",
  "profile_complete": true,
  "preferred_language": "ru",
  "created_at": "2026-01-10T10:00:00+05:00"
}

    PUT /api/auth/profile/¶

    Обновить профиль. profile_complete пересчитывается автоматически.

    Права: IsAuthenticated

    // Request (partial — отправлять только изменяемые поля)
{
  "last_name": "Иванов",
  "first_name": "Иван",
  "patronymic": "Иванович",
  "phone": "+7 777 123 45 67",
  "email": "ivan@example.com",
  "country": "Казахстан",
  "city": "Алматы",
  "preferred_language": "ru"
}

// Response 200 — обновлённый профиль (та же схема, что GET)

    POST /api/auth/set-role/¶

    Установить роль пользователя (parent или specialist). Используется при первом входе.

    Права: IsAuthenticated

    // Request
{ "role": "parent" }

// Response 200
{ "role": "parent", "profile_complete": false }

// Response 400
{ "role": ["\"admin\" is not a valid choice."] }

    POST /api/auth/request-code/ (legacy)¶

    Вход через Telegram username. Не использовать в основном frontend flow.

    // Request
{ "username": "@example_user" }

// Response 200  { "status": "code_sent", "expires_in_seconds": 600 }
// Response 404  { "error": "Telegram user not found" }
// Response 429  { "error": "Too many attempts. Try again in 5 minutes." }

// Response 200  { "access": "...", "refresh": "...", "is_new_user": true, "profile_complete": false }

    3. Скрининг (/api/screening/)¶

    GET /api/screening/modules/¶

    Список активных скрининговых модулей.

    Права: IsAuthenticated
    Query: ?child_age=4 — фильтр по возрасту ребёнка в месяцах

    // Response 200
[
  {
    "id": 1,
    "name_ru": "BERKANA IORR v3.3: 0-6 месяцев",
    "name_en": "BERKANA IORR v3.3: 0-6 months",
    "name_kz": "BERKANA IORR v3.3: 0-6 ай",
    "description_ru": "Research Use Only...",
    "age_min_months": 0,
    "age_max_months": 6,
    "rescreen_interval_days": 60,
    "questions_count": 56
  }
]

    Статус: Импортирован только модуль для 0–6 месяцев. Для детей старше 6 месяцев запрос вернёт пустой массив.

    GET /api/screening/modules/{module_id}/questions/¶

    Все активные вопросы модуля.

    Права: IsAuthenticated
    Query: ?child_age=4 — фильтрует по age_min_months / age_max_months

    // Response 200
[
  {
    "id": 1,
    "code": "M1",
    "block": "🟢 БЛОК 2: КРУПНАЯ МОТОРИКА",
    "domain": "motor_gross",
    "question_type": "berkana_4opt",
    "title_ru": "Приподнимает голову лёжа на животе",
    "title_en": "Lifts head while lying on stomach",
    "title_kz": "...",
    "text_ru": "Положите ребёнка на живот...",
    "help_ru": "Подъём головы показывает...",
    "options": [
      {"value": "yes",       "label_ru": "ДА",          "score": 10},
      {"value": "sometimes", "label_ru": "ИНОГДА",      "score": 5},
      {"value": "not_yet",   "label_ru": "ЕЩЁ НЕТ",    "score": 0},
      {"value": "not_tried", "label_ru": "НЕ ПРОБОВАЛИ","score": null}
    ],
    "age_min_months": 2,
    "age_max_months": 6,
    "include_in_score": true,
    "sort_order": 10
  }
]

    Домены: motor_gross, motor_fine, speech, social, cognitive, sensory, red_flag, early_reg
    Типы вопросов: berkana_4opt (основной), yes_no, scale_1_5
    Скоринг berkana_4opt: yes→10, sometimes→5, not_yet→0, not_tried→null (исключается из знаменателя)

    POST /api/screening/sessions/¶

    Создать новую или вернуть существующую in_progress сессию для ребёнка.

    Права: IsAuthenticated (родитель-владелец ребёнка)

    // Request
{ "child": 1 }

// Опционально — указать модуль явно
{ "child": 1, "module": 1 }

// Response 201 (новая) / 200 (resumed)
{
  "id": 10,
  "child": 1,
  "child_name": "Иванов Иван",
  "module": 1,
  "module_name": "BERKANA IORR v3.3: 0-6 месяцев",
  "status": "in_progress",
  "started_at": "2026-05-04T20:34:00+05:00",
  "completed_at": null,
  "next_rescreen_at": null,
  "questions_count": 56,
  "answered_count": 0
}

// Response 400 — нет подходящего модуля
{ "detail": "No active screening module is available for this child age." }

    Состояние сессии со всеми сохранёнными ответами.

  "questions_count": 56,

    {
      "id": 1,
      "question": 5,
      "value": "yes",
      "score": 10,
      "answered_at": "2026-05-04T20:36:00+05:00"
    }
  ]
}

    GET /api/screening/sessions/{id}/questions/¶

    Вопросы сессии, отфильтрованные по возрасту ребёнка.

    Права: IsAuthenticated + IsSessionOwner
    Возвращает ту же схему, что GET /screening/modules/{id}/questions/.

    POST /api/screening/sessions/{id}/answers/¶

    Upsert ответа. Если ответ на вопрос уже был — перезаписывается.

    Права: IsAuthenticated + IsSessionOwner

    // Request
{ "question": 5, "value": "sometimes" }

// Response 200
{
  "id": 2,
  "question": 5,
  "value": "sometimes",
  "score": 5,
  "answered_at": "2026-05-04T20:37:00+05:00"
}

    POST /api/screening/sessions/{id}/complete/¶

    Завершить сессию. Считает next_rescreen_at, создаёт Report.

    Права: IsAuthenticated + IsSessionOwner

    // Response 200
{
  "id": 10,
  "status": "completed",
  "completed_at": "2026-05-04T20:45:00+05:00",
  "next_rescreen_at": "2026-07-03T20:45:00+05:00",
  "report_id": 7
}

// Response 400
{ "detail": "Not all required questions are answered.", "missing_count": 12 }

    POST /api/screening/sessions/{id}/feedback/¶

    Сохранить ответы на опрос обратной связи (5 вопросов после скрининга).

    Права: IsAuthenticated + IsSessionOwner

    // Request
{
  "q1": true,
  "q2": true,
  "q3": false,
  "q4": true,
  "q5": true
}

// Response 200
{ "status": "ok" }

    GET /api/screening/reports/{id}/pdf/¶

    Скачать PDF отчёта.

    Права: IsAuthenticated (владелец)
    Response 200: Content-Type: application/pdf

    GET /api/screening/reports/{id}/exercises-pdf/¶

    Скачать PDF с упражнениями по доменам в жёлтой/красной зоне.

    Права: IsAuthenticated (владелец)
    Response 200: Content-Type: application/pdf
    Response 204: Все домены в норме, упражнений нет

    5. Валидационные апоинтменты (/api/validation/)¶

    Валидационный flow: после скрининга с overall_level = follow_up отчёт может быть направлен к специалистам-консультантам через ValidationAppointment.

    cancelled Отменён

    Импорт скрининговых модулей¶

    Admin REST API (рекомендуемый способ)¶

    Права: role=admin. Двухфазный процесс через Django FileBasedCache (TTL 30 мин).

    GET /api/admin/screening/template/¶

    Скачивает .docx шаблон для заполнения. Response: application/vnd.openxmlformats-officedocument.wordprocessingml.document.

    POST /api/admin/screening/import/preview/¶

    Body: multipart/form-data { file: .docx }. Парсит файл, возвращает preview без записи в БД.

    {
  "session_id": "uuid4",
  "module_info": { "name_ru": "...", "name_en": "...", "name_kz": "...", "age_min": 0, "age_max": 6, "rescreen_interval_days": 90 },
  "questions_count": 56,
  "rules_count": 12,
  "questions": [{ "code": "M1", "domain": "motor_gross", "type": "berkana_4opt", "text_ru": "...", "has_en": true, "has_kz": true }],
  "scoring_rules": [{ "domain": "motor_gross", "severity": "typical", "min_score": 75, "max_score": 100 }],
  "warnings": ["Вопрос M3: help_en пустой"],
  "errors": []
}
    • errors пустой → 200 OK
    • errors непустой → 422 Unprocessable Entity (preview всё равно возвращается)

    POST /api/admin/screening/import/confirm/¶

    Body: { "session_id": "uuid4" }. Записывает данные в БД через update_or_create (идемпотентен).

    { "imported_questions": 56, "imported_rules": 12, "module_name": "BERKANA IORR v3.3: 0-6 месяцев", "module_id": 3 }
    • 404 — сессия истекла (> 30 мин)
    • 400 — ParseResult содержит errors (нельзя импортировать)
